# IT consulting firm

A starter project for a boutique IT consulting firm (architecture, delivery, devops, security). Six lanes covering the engagement lifecycle.

## What's in this template

| Directory | What lives here | Agent role |
|---|---|---|
| `presales-discovery/` | Lead qualification, discovery sessions, proposals, SOWs | Pre-sales lead |
| `architecture-design/` | Solution architecture, technology selection, decision records, design reviews | Solution architect |
| `delivery/` | Active engagements: backlog, sprint state, status reports, risks | Delivery lead |
| `devops-platform/` | Cloud accounts, infra-as-code, CI/CD, environments, observability | Platform engineer |
| `security-compliance/` | Security posture, SOC2/ISO27001 evidence, pen test schedule, customer due diligence questionnaires | Security officer |
| `client-success/` | Customer health, expansion, renewal, advocacy | Client success manager |

## Tier-based provisioning

- **Tier 1:** all 6 agents auto-start.
- **Basic:** `delivery/` + `presales-discovery/` is the typical pair for a 1–3 person firm.

## Suggested first steps

1. **`presales-discovery/`** — capture your standard discovery questions, your SOW template, your rate card.
2. **`delivery/`** — register active engagements with scope, milestones, billing model.
3. **`architecture-design/`** — feed in your standard reference architectures + technology preferences.
4. **`devops-platform/`** — load your cloud account inventory + access model.
5. **`security-compliance/`** — current security posture + customer DDQs you've answered.
6. **`client-success/`** — health-check cadence per customer.

## Customising

- Add lanes for your speciality: `data-analytics/`, `mobile/`, `embedded/`, `ml-ai/`, `industry-vertical/`.
- Rewrite each agent's README to match how YOUR firm works (Scrum vs Kanban, fixed-price vs T&M, etc.).
